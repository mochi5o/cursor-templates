# API Design Skill

## このスキルの役割
- RESTful API の設計レビューと改善提案を支援する

## 前提知識
- RESTful API 設計原則
- HTTP メソッドとステータスコードの適切な使用
- バージョニング戦略
- エラーハンドリング

## 判断基準
- RESTful 原則に従っているか
- 適切な HTTP メソッドとステータスコードを使用しているか
- API バージョニングが適切に実装されているか
- エラーレスポンスが統一されているか
- リクエスト/レスポンスの形式が一貫しているか

## よくある改善提案
- リソース指向の URL 設計
- 適切な HTTP ステータスコードの使用（200, 201, 400, 401, 404, 500 等）
- API バージョニングの実装（/api/v1/ 等）
- 統一されたエラーレスポンス形式
- ページネーションの実装
- フィルタリング・ソート・検索機能の提供
