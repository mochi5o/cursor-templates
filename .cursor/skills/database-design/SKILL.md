# Database Design Skill

## このスキルの役割
- データベース設計のレビューと改善提案を支援する

## 前提知識
- リレーショナルデータベース（PostgreSQL, MySQL 等）
- 正規化理論
- インデックス設計
- マイグレーション戦略

## 判断基準
- テーブル設計が適切に正規化されているか
- インデックスが適切に設定されているか
- 外部キー制約が適切に設定されているか
- N+1 問題を引き起こす可能性がないか
- マイグレーションが安全に実行できるか

## よくある改善提案
- 適切な正規化レベル（過度な正規化を避ける）
- 頻繁にクエリされるカラムへのインデックス追加
- 外部キー制約による参照整合性の確保
- パーティショニングの検討（大規模データの場合）
- マイグレーションのロールバック戦略
