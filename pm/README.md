# PM向けCursorテンプレート

このディレクトリには、プロダクトマネージャーがCursorを使い始めるためのテンプレートが含まれています。

## 含まれる内容

### Rules（ルール）
- `pm-spec.mdc` - 仕様管理のルール
- `user-story.mdc` - ユーザーストーリー記述のルール

### Commands（コマンド）
- `create-spec.md` - 新規機能の仕様書作成
- `update-spec.md` - 既存仕様書の更新
- `spec-review.md` - 仕様書のレビュー
- `stakeholder-alignment.md` - ステークホルダー間の認識合わせ

### Skills（スキル）
- `pm-spec-review/` - 仕様書レビューの専門知識
- `user-story-writing/` - ユーザーストーリー作成のベストプラクティス
- `acceptance-criteria/` - 受け入れ基準の定義方法

### Agents（サブエージェント）
- `pm-alignment-subagent.md` - 仕様・要件の整理と認識合わせ
- `requirements-gathering-subagent.md` - 要件収集と整理

### テンプレート
- `docs/templates/spec-template.md` - 仕様書テンプレート
- `docs/templates/user-story-template.md` - ユーザーストーリーテンプレート

### その他
- `.cursorignore` - Cursorが読み取らないファイルを指定

## 使い方

1. このディレクトリの内容をプロジェクトルートにコピー
2. `docs/` ディレクトリ構造を参考にドキュメントを整理
3. 必要なCommands、Skills、Agentsを選択して使用

詳細は [README-PM.md](./README-PM.md) を参照してください。
